E-Commerce SQL Analytics Project

About the Project
This project is a simple e-commerce database I built using SQL to understand how real business data is stored and analyzed. The main goal was to practice writing queries and get insights like customer spending, product sales, and revenue trends.

What I Did
* Created tables for customers, products, orders, and order details
* Connected the tables using primary key and foreign key
* Inserted sample data to simulate real-world transactions

Analysis I Performed
* Found the top customers based on total spending
* Calculated monthly revenue to see sales trends
* Identified best-selling products
* Checked which customers placed multiple orders
* Calculated revenue generated by each product category

Concepts I Used
* **JOIN** → to combine data from multiple tables
* **GROUP BY** → to summarize data
* **HAVING** → to filter grouped results
* **Window Functions** → to calculate running total
* **RANK()** → to rank products based on sales

Tools Used
* MySQL (DB Fiddle for execution)

What I Learned
* How to design a basic relational database
* How different tables are connected in real systems
* Writing SQL queries to get useful business insights
* Understanding how data is analyzed in backend

Project Structure
schema.sql → contains table creation and data insertion
queries.sql → contains all analysis queries
screenshots → output of queries

Note
This project helped me understand how SQL is used in real-world scenarios, especially for analyzing business data.
